# Beautify the RGB Sunglasses companion app (issue #40)

## Context

The companion app (`app/`, Expo SDK 54 / RN 0.81 / React 19, expo-router) is the unmodified Expo
starter template with BLE features bolted on. It looks unfinished: the default `ThemedText`/`ThemedView`
primitives, hard-coded colors that ignore the theme (`#fff` text, `#ccc` borders, `#4499ff`/`#4CAF50`
progress bars), native `<Button>`s in modals, a leftover **React-logo parallax header** on the connect
screen, and a dead "Home" welcome tab. Issue #40 asks to make it _beautiful, fast, easy to use_ while
respecting OS light/dark mode.

**Decided with the user:** (1) a **bespoke design system** built on the app's existing theme mechanism —
no Paper / NativeWind / Tamagui (Paper has a reported iOS hang on this exact SDK 54 + React 19 stack);
(2) a **vibrant "RGB" brand identity** (violet → magenta → cyan, gradients used sparingly);
(3) **freedom to rethink the information architecture**, as long as no feature is lost.

**Outcome:** a cohesive token-driven UI, a small reusable component kit, a cleaner connect-first navigation,
and themed light/dark across every screen — with all BLE behavior and the Jest suite preserved.

### Theme plumbing

- In `app/hooks/use-theme-color.ts`, add a convenience hook next to the existing `useThemeColor`:
  ```ts
  export function useThemeColors() {
    const scheme = useColorScheme() ?? "light";

  }
  ```

  device-state "Update", any modal link-buttons). Verify those still navigate.
- Don't give `AppButton` an internal spinner — it would break the device-list-item `ActivityIndicator` assertions.
- Keep the boolean control a native `Switch` (role `switch`) and dropdown options as `ThemedText` (matched by text).
- `reactCompiler: true`: read theme via hooks at render (`useThemeColors()`), don't mutate shared style objects.
- Gradients: keep small text off the cyan stop; labels ≥16px/≥600 weight for AA-large.
